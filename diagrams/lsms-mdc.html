lsms ()

8개 위성 프로젝트 × 엔지니어 70명의 리소스 배정 플랫폼. 레이어 간 의존 방향이 한쪽으로만 흐르는 Module → Domain → Context 3-Layer 패턴을 도입.
01.레이어 정의 — 의존은 언제나 한 방향
flowchart LR subgraph C["Context Layer (UseCase)"] UC1["AssignResourceUseCase"] UC2["ApproveScheduleUseCase"] UC3["ResolveConflictUseCase"] end subgraph D["Domain Layer (Aggregate)"] A1["Project
(aggregate root)"] A2["Resource"] A3["Assignment"] A4["Schedule"] end subgraph M["Module Layer (Bounded Context)"] M1["project-module"] M2["resource-module"] M3["assignment-module"] M4["schedule-module"] M5["notification-module"] end UC1 -->|uses| A1 UC1 -->|uses| A2 UC1 -->|uses| A3 UC2 -->|uses| A4 UC3 -->|uses| A3 A1 -->|hosted in| M1 A2 -->|hosted in| M2 A3 -->|hosted in| M3 A4 -->|hosted in| M4 classDef ctx fill:#1d3a1f,stroke:#39ff14,color:#d4ffd4; classDef dom fill:#102012,stroke:#49a83a,color:#d4ffd4; classDef mod fill:#0c1109,stroke:#4a7a4c,color:#9ef29e; class UC1,UC2,UC3 ctx; class A1,A2,A3,A4 dom; class M1,M2,M3,M4,M5 mod;
→ Context는 Domain만 부른다. Domain은 Module만 부른다. 반대 방향은 컴파일/린트 단계에서 차단.
→ 새 기능은 Context 한 개 추가만으로 끝나는 일이 많아짐 → 리그레션 면적이 작다.
02.핵심 유스케이스 — "리소스 이중 배정 방지"
sequenceDiagram autonumber participant UI as Web UI participant UC as AssignResourceUseCase (Context) participant P as Project (Domain) participant R as Resource (Domain) participant A as Assignment (Domain) participant REPO as AssignmentRepo (Module) participant BUS as Event Bus participant NOTI as notification-module UI->>UC: assign(projectId, resourceId, slot) UC->>P: load(projectId) UC->>R: load(resourceId) UC->>A: new Assignment(project, resource, slot) A->>A: invariant check
(슬롯 중복 · 역량 매칭) alt conflict A-->>UC: DomainError("slot occupied") UC-->>UI: 409 + 대체 슬롯 제안 else ok UC->>REPO: save(assignment) REPO->>BUS: assignment.created BUS->>NOTI: 당사자/PM에 통지 UC-->>UI: 200 OK end
03.알림 DSL — 채널 · 시간대 · 선호를 한 표현으로
when event.assignment.created
and user.role in ["pm", "assignee"]
deliver via first-of(
  channel.fcm      where user.prefers("push") and now.in(user.workHours),
  channel.webpush  where user.online,
  channel.email    otherwise
)
quiet-hours 22:00..07:00 (KST)
→ DSL이 먼저, UI가 그 뒤. 사용자는 채널 우선순위를 직접 배치, 개발자는 구현을 추상화 뒤에 숨김.
